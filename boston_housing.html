Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

For the performance_metric function in the code cell below, you will need to implement the following:

  • Use r2_score from sklearn.metrics to perform a performance calculation between y_true and y_predict.

# Import 'r2_score'
from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    # Calculate the performance score between 'y_true' and 'y_predict'
    score = r2_score(y_true, y_predict)
    
    # Return the score
    return score

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer: The benefit of splitting a dataset into training and testing subsets is the ability to test if our model generalizes well enough. The important thing to keep in mind when developing a learning algorithm is that a model should be able to perform well on new, previously unobserved data. We hold out a subset of our data to test the model's performance and never use it on a training stage. The reasonable ratio of train and test size allows to avoid underfitting and overfitting: if a train set is not large enough, it is harder to train a model to capture the dependencies, but if we use all of the data available, there is a big chance that a model simply memorizes all the data points and won't be able to generalize well.

Question 7 - Grid Search¶

  • What is the grid search technique?
  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer:

  • Grid search is a technique of choosing a best performing set of hyperparameter values from all the possible combinations of values provided to the algorithm. Grid search allows to choose optimal hyperparameters automatically instead of picking them manually.
  • Assume that one has chosen an algorithm to use, e.g. a decision tree. The problem that arises is how to choose optimal parameters of the model. One can consider varying values of maximum depth and minimum number of samples in leaf nodes. Then one defines the ranges of values of hyperparameters, e.g. maximum depth is from 1 to 10 and minimum number of samples in leaves are even numbers from 2 to 20. The training dataset is split into a training and a validation subsets. Then a model is fit for each combination of the parameters on the training subset. One can think of an N-dimension table (a grid) where N is a number of parameters and each of the N axes is marked up with values of a corresponding paramer within the given range. Each model is then validated on a validation subset using a desired metric and values of the metric are written to the corresponding cells of the grid. Finally, a combination of parameters corresponding to the best value of the metric is chosen. This combination can be considered optimal hyperparameter values.

Answer:

  • k-fold cross-validation is a technique to validate a trained algorithm. Initial training set is split into k subsets and a model is trained k times. Each time k-1 subsets are used for training and remaining subset is used for validation.
  • k-fold cross-validation can be used in combination with grid search. When running an ordinary grid search we usually have our entire dataset split into three subsets: training set, validation set and testing set. This way we have reduced the size of the dataset used for fitting the model and there is a chance to underfit or to depend on the random split of the training dataset. With k-fold cross-validation we actually can use the whole training set for fitting and validating the model and obtain more accurate validation scores as the model is fit and validated on different subsets each of the k times. This technique makes validation less dependent on random splits of the training set.

# Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y, random_state=0):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = random_state)

    # Create a decision tree regressor object
    regressor = DecisionTreeRegressor(random_state=random_state)

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':list(range(1,11))}

    # Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
